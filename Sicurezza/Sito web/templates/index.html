🎓 Cosa sta succedendo? (IDOR / Broken Access Control)

Il server ti mostra i dati basandosi unicamente sul parametro user_id presente nell'URL (la barra degli indirizzi del tuo browser).

Il problema nel codice (app.py):

user_id = request.args.get('user_id', '1')
user_data = USERS_DB.get(user_id)

Il server non si fa la domanda cruciale: "La persona che sta richiedendo l'ID 3, è davvero autorizzata a vedere l'ID 3?". Si fida ciecamente dell'input.

Prova ad hackerarlo!

Guarda l'URL in alto nel tuo browser. Probabilmente finisce con / o /?user_id=1.

Prova a modificare manualmente l'URL aggiungendo o cambiando il parametro in /?user_id=2 o /?user_id=3 e premi Invio. Riuscirai a leggere i segreti degli altri utenti e perfino a rubare la "Flag" dell'amministratore!
